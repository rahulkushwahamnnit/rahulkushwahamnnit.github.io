FPGA Design Contest requires one to write the code for the given problem statement in either VHDL or Verilog and optimize it, then dump the code on the FPGA Kit to physically realize the design that was coded. Problem Statement

The problem requires the design of a Single Player Shooter Game. It will be played on an LED matrix. The circuit must be designed without using any Programmable device.

Detailed Description

The arena consists of an LED matrix. At one end there will be the Terminator. Zombies will enter the arena from the other end (min. 4 in a row and randomly) and will move down the arena (upto 3 rows of LED matrix). You have to generate a 4 bit(atleast) random number sequence. This number is used as position of zombies in first row.Now after every clock pulse these positions will shift to the row below, up to 3 rows (min) and the topmost row will be taken by a new number generated .The position of the terminator can be controlled by buttons representing Left and Right keys. Function of the terminator is to kill the zombies. This you can do by clearing the LED which is in front of the shooter position as fast as possible. If zombies succeed in crossing the lowest row (atleast 3) then, the game will terminate. This will be indicated by a red indiactor LED. If after a certain interval of time, no zombies are able to cross the last row then a green indicator LED will be glown.


Add Ons:

Following features in your design can fetch you extra points and will be used to resolve ties.
• Randomness in the zombies position in the first row.
• Make an alarm bell ring when the indicator LED turns red.
• Bigger size of LED matrix. (min 4X3).

Rules and Regulations

• This is a team event and is Exclusively for 1st and 2nd year.

• Use of any programmable device is strictly prohibited.

• A team can consists of maximum of 3 memebers (and atleast one of which must belong to Electrical or ECE branch).

• Decision of Judges will be final and binding.

• Abstract Submission:
   An abstract in the form of powerpoint presentation must be submiited by the participating teams. The abstract must contain the basic understanding of the problem statement along with an approximate solution.

Judging Criteria

• Simplicity of algorithm is appreciated.
